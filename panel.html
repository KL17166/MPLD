👑 Modo VIP — Interceptador XHR/Fetch

Diferente das regras normais (que atuam no DOM visível), o Modo VIP age diretamente nas respostas de rede antes que o JavaScript da página as processe. Ideal para dashboards, apps React/Vue e dados carregados via AJAX.

👑

Interceptação de Requisições de Rede

Injeta um monkey-patch em XMLHttpRequest e window.fetch() na página. Cada resposta JSON passa pelos seus filtros antes de ser entregue ao código do site, permitindo alterar valores como preços, saldos e labels em APIs não acessíveis pelo DOM.

Desativado
🔄

Intercepta XHR & Fetch

Sobrescreve os métodos nativos do browser. Captura toda requisição assíncrona da página, independente da biblioteca usada (axios, jQuery, etc.).

⚡

Antes do Processamento

Os valores são modificados na raiz, antes do JSON.parse(). O React/Vue nunca vê o dado original — recebe direto o valor substituído.

🛡️

Anti-Recarregamento

Perfeito para Single Page Apps (SPAs) que atualizam dados constantemente via polling ou WebSocket, sem recarregar o HTML.

💡
Como usar o Modo VIP: Ative o toggle acima. Depois, ao criar ou editar uma regra na aba Regras, marque "Ativar Network Hooker (Fetch/XHR)". Essas regras exibem a coroa 👑 no card.

Limitação: Funciona apenas com respostas de texto (JSON, HTML, XML). Binários (imagens, vídeos) são ignorados.

📱 Como configurar em Dispositivos Externos (Celular/Tablet)

Para interceptar apps mobile, configure o dispositivo para passar pelo mesmo proxy MITM na sua rede local.

🤖 Android
  1. Conecte à mesma rede Wi-Fi do computador.
  2. Editar rede → Proxy manual → Host: IP_DO_PC, Porta: 8080
  3. No Chrome do Android, acesse o IP acima na porta 8888 e baixe o certificado CA.
  4. Configurações → Segurança → Instalar certificado de CA.
🍎 iOS (iPhone / iPad)
  1. Editar Wi-Fi → Proxy manual → mesmo IP e porta acima.
  2. Safari → baixar certificado → será criado um "Perfil Baixado".
  3. Ajustes → Instalar Perfil.
  4. Ajustes → Geral → Sobre → Confiança de Certificados → ativar a chave do proxy.
🔒 Apps com Certificate Pinning (Android)

Apps com pinning rejeitam o CA externo. Solução: recompilar o APK aceitando CAs customizadas.

npm install -g apk-mitm
apk-mitm app.apk

Configurações Comportamentais

Controla quando e como a extensão aplica as substituições nas páginas que você visita. Ajuste conforme o tipo de site — sites dinâmicos (React, Vue) precisam de estratégias diferentes de sites estáticos.

Aplicação Automática (DOM Init)

Executa todas as regras ativas imediatamente no evento window.onload da página. Ideal para sites estáticos ou com pouco conteúdo dinâmico. Se o conteúdo aparecer depois do carregamento inicial, use também a opção "Observar Mudanças no DOM" abaixo.

Observar Mudanças no DOM (MutationObserver)

Instancia um MutationObserver que escuta toda vez que novos nós HTML são inseridos na página. Essencial para sites de scroll infinito, cliques que carregam mais conteúdo, e Single Page Apps (SPAs) como Gmail, Twitter e dashboards que atualizam sem recarregar a página.

🔄 Modo Contínuo (Polling Forçado)

Executa as substituições em loop usando setInterval, independentemente de eventos DOM. Útil para sites que limpam o MutationObserver ao reescrever o innerHTML inteiro (ex: dashboards baseados em WebSocket pesado, ou apps React que remontam componentes continuamente). Atenção: pode aumentar uso de CPU. Prefira o MutationObserver quando possível.

Intervalo do Polling Contínuo

Define de quantos em quantos milissegundos o loop de substituição é executado. Valores menores = mais agressivo (mais CPU). Valores maiores = mais econômico, mas pode deixar o conteúdo original visível por frações de segundo.
Recomendado: 500ms para a maioria dos casos.

⚠️ Apagar Todos os Dados (Hard Reset)

Executa chrome.storage.local.clear() apagando permanentemente todas as regras, configurações, histórico e estatísticas. Essa ação não pode ser desfeita. Considere exportar suas regras antes de prosseguir.

Estatísticas de Execução

Relatório acumulado de quantas vezes cada regra foi aplicada com sucesso. Os dados são coletados pelos Content Scripts e enviados ao Background Service Worker. Útil para identificar quais regras estão sendo mais ativas.

Nova Regra de Substituição

Preencha os campos abaixo para criar uma substituição automática.

Use um nome descritivo para identificar a regra na lista. Se deixar vazio, será gerado automaticamente.
Com Regex ativo, use padrões como \d+\.\d{2} para encontrar valores numéricos.
Com Regex, use $1, $2 para grupos de captura.
Restringe a regra a URLs específicas. Suporta texto simples (substring match) ou Regex. Deixe vazio para aplicar em todos os sites.
A regra age diretamente nos elementos de texto da página (substituição visual via TreeWalker).
